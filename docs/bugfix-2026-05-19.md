## Bugs Encontrados e Corrigidos

### BUG-01 — Nome de função obsoleto após renomear rota
- **Arquivo:** `app/menu/page.tsx`
- **Problema:** Função exportada ainda se chamava `DemoPage` após a rota `/demo` ser renomeada para `/menu`
- **Correção:** Renomeado para `MenuPage`

### BUG-02 — Delay artificial de 900ms no checkout
- **Arquivo:** `components/cardapio/CheckoutForm.tsx:46`
- **Problema:** `await new Promise(r => setTimeout(r, 900))` antes de enviar o pedido causava 900ms de espera artificial na UX e atrasava o feedback de erro para o usuário
- **Correção:** Linha removida. O spinner já é exibido via `loading = true`

### BUG-03 — Acento faltando no texto do carrinho vazio
- **Arquivo:** `components/cardapio/Carrinho.tsx:83`
- **Problema:** "Seu pedido ainda **esta** vazio." — falta acento
- **Correção:** "Seu pedido ainda **está** vazio."

### BUG-04 — Botão fechar usa "x" ASCII em vez de "×" tipográfico
- **Arquivos:** `components/cardapio/Carrinho.tsx:69`, `components/cardapio/PratoModal.tsx:39`
- **Problema:** Caractere `x` (letra minúscula) usado como ícone de fechar. Visualmente inferior e semanticamente incorreto
- **Correção:** Substituído por `×` (U+00D7, sinal de multiplicação tipográfico)

### BUG-05 — Payload do checkout enviava dados desnecessários do cliente
- **Arquivo:** `components/cardapio/CheckoutForm.tsx`
- **Problema:** O payload enviava `restauranteNome`, `nome`, `preco` (por item) e `total` — dados que o servidor ignora completamente (recalcula tudo do catálogo). Aumentava o payload, criava superfície de ataque desnecessária e poderia confundir desenvolvedores futuros
- **Correção:** Payload reduzido ao mínimo: `restauranteId`, `cliente`, e por item apenas `pratoId` + `quantidade`

### BUG-06 — Type `PedidoSimuladoPayload` aceitava campos ignorados pelo servidor
- **Arquivo:** `types/index.ts`
- **Problema:** Interface definia `restauranteNome?`, `nome?`, `preco?` e `total?` — campos que o servidor nunca lê, criando falsa impressão de que eram usados
- **Correção:** Interface enxuta com apenas os campos que o servidor efetivamente processa
